Protection Modes

Choose a starting mode, then fine-tune each protection. Changes are saved to extension storage and applied to new page reads without restarting the browser.

Feature Toggles

Header protections only remove `ETag` and `If-None-Match`, and they are off by default to avoid unnecessary cache breakage. Screen normalization is intended for Strict mode. Turning it on can break page layouts and create contradictions with the browser window size, so it stays off in Standard.

Strict mode allows WebGL renderer spoofing, but rare fake renderers can increase uniqueness. Keep Windows-only WebGL profiles paired with a Windows User-Agent/platform.

Navigator Profile

Hardware concurrency and device memory stay configurable without changing the User-Agent string.
